FLYNEXT TRAVEL APP - Comprehensive Project Plan

Executive Summary

FlyNext is a streamlined flight booking platform designed to facilitate seamless travel planning for users and data-driven revenue tracking for administrators. The application integrates Google OAuth for secure authentication and Stripe for robust payment processing, allowing travelers to search, book, and pay for flights within a unified interface. By providing real-time booking metrics, the platform ensures management has clear visibility into business performance and growth.

The system architecture utilizes a high-performance stack featuring a Next.js frontend and a FastAPI backend, supported by a PostgreSQL database. To ensure scalability and operational consistency, the application is containerized via Docker and deployed to AWS ECS, leveraging S3 for object storage. This cloud-native approach provides a resilient, production-ready environment capable of handling concurrent travel transactions and secure data management.

1. Requirements & User Stories

Functional Requirements

  • User authentication via Google OAuth
  • Search for flights by date and destination
  • Book and pay for flights using Stripe

User Stories

  • As a Traveler, I want to search for flights so that plan a vacation.
  • As a Admin, I want to view booking metrics so that track revenue.

2. System Architecture

Tech Stack

  • frontend: Next.js + Tailwind CSS
  • backend: FastAPI (Python)
  • database: PostgreSQL
  • devops: Docker + AWS S3
